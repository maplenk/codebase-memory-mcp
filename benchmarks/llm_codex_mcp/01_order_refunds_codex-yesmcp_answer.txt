Refund amount calculation is a mix of client-supplied refund payload and server-side reconciliation:
- Total refund amount is `sum(refundPayments[].amount)`. It is added into `orderSummary.voidAmount`, and the server rejects the request if that would push original `grossBill` below zero. [Order.php](/Users/naman/Documents/QBApps/qbapi/app/Order.php#L15952)
- Line-item void amount is proportional to the original line: `(old productValue / old quantityOrdered) * voidQuantity`, accumulated into `orderDetail.voidAmount`. [Order.php](/Users/naman/Documents/QBApps/qbapi/app/Order.php#L15598)
- Refund invoice rows are created as negative deltas between original and post-refund values for `netBill`, `productValue`, `taxableValue`, `discountTotalValue`, `totalTaxValue`, `totalChargeValue`, etc. Full cancel and partial cancel diverge here. [Order.php](/Users/naman/Documents/QBApps/qbapi/app/Order.php#L16192)
- Loyalty is recomputed from the reduced gross/net values; customer points and loyalty ledger are updated accordingly. [Order.php](/Users/naman/Documents/QBApps/qbapi/app/Order.php#L15157)
- Online pocket refunds use accounting entries, not UI payment rows: `OnlineOrder::partiallyRefundEcomOrder` sums `ACC_POCKET_RECD` credits from `orderAccounting` for the refund order. [OnlineOrder.php](/Users/naman/Documents/QBApps/qbapi/app/OnlineOrder.php#L11402)

**Integrations And Edge Cases**
- E-commerce pocket refunds are async: main refund flow sets `isOnlinOrder` when a refund payment is `PAYMENT_POCKET`, controller dispatches `EcomVoidOrderPaymentRefundJob`, and that job calls `OnlineOrder::partiallyRefundEcomOrder`. [OnlineOrder.php](/Users/naman/Documents/QBApps/qbapi/app/OnlineOrder.php#L4922) [OnlineOrder.php](/Users/naman/Documents/QBApps/qbapi/app/OnlineOrder.php#L4987) [OnlineOrder.php](/Users/naman/Documents/QBApps/qbapi/app/OnlineOrder.php#L5023)
- Direct gateway refund paths are separate from the core refund flow: Razorpay void via `thirdPartyPaymentIntent`, SnapMint refund with duplicate/cumulative checks, Paytm Cloud refund with status mapping, and WorldLine/PayPhi full invoice refund or void. [razorPay.php](/Users/naman/Documents/QBApps/qbapi/app/razorPay.php#L436) [snapMint.php](/Users/naman/Documents/QBApps/qbapi/app/snapMint.php#L504) [paytm.php](/Users/naman/Documents/QBApps/qbapi/app/paytm.php#L784) [PayPhiWorldLine.php](/Users/naman/Documents/QBApps/qbapi/app/PayPhiWorldLine.php#L369)
- Partial refunds are supported with `ORST_PARTIAL_CANCELLED`; full refunds use `ORST_CANCELLED`.
