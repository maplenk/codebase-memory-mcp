BNPL is not a single enum change in this codebase. Payment types are mostly data-driven from DB tables, but several order, export, and report paths still hardcode known `PAYMENT_*` values.

**Where Payment Types Are Defined**
- `[app/Order.php](/Users/naman/Documents/QBApps/qbapi/app/Order.php#L2554)` `getChainPaymentTypes()` reads payment modes from `globalTypeValues` (`typeID = 7`) plus `globalPaymentAccountMapping`.
- `[app/Admin.php](/Users/naman/Documents/QBApps/qbapi/app/Admin.php#L7213)` manages store-level enablement in `store{storeID}_paymentSettings`; `addStorePaymentSettings()` also requires a matching `globalPaymentAccountMapping` row.
- `[app/Http/Controllers/v2/SetupWeb.php](/Users/naman/Documents/QBApps/qbapi/app/Http/Controllers/v2/SetupWeb.php#L506)` exposes the add/update/get APIs for store payment settings.
- `[app/QBChannel.php](/Users/naman/Documents/QBApps/qbapi/app/QBChannel.php#L104)` exposes payment modes for channel mappings from `globalTypeValues` and stores selections in `chain{chainID}_channelPaymentMap`.
- `[scripts/helpers/convertPI.js](/Users/naman/Documents/QBApps/qbapi/scripts/helpers/convertPI.js#L297)` builds `paymentList` from `paymentSettings` plus global mappings for PI/online flows.
- `[app/Friday.php](/Users/naman/Documents/QBApps/qbapi/app/Friday.php#L729)` copies payment settings across stores.

You will need DB/config changes even if you touch no PHP:
- Add BNPL to `globalTypeValues` with `typeID = 7`
- Add a `globalPaymentAccountMapping` row for BNPL
- Enable BNPL in each `store{n}_paymentSettings`
- If channels use it, add it to `chain{n}_channelPaymentMap`

- `[app/Order.php](/Users/naman/Documents/QBApps/qbapi/app/Order.php#L7930)` declares the main payment tables: store `orderPayments`, chain `orderPayments`, chain `invoicePayments`, plus `orderAccounting`.

Likely no schema migration is needed for basic BNPL because these tables already store generic `paymentType`, `transactionID`, `referenceID`, `approvalCode`, `partyName`, etc. If you need BNPL-specific metadata like lender, plan, tenure, or due date, then add columns to `orderPayments`, `invoicePayments`, and probably `creditPaymentDetail`.
